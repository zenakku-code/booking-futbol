# SaaS Football Booking - Roadmap 🚀

This document outlines the strategic growth plan for the SaaS platform.

## Phase 1: Foundation & Security (COMPLETED)
- [x] Multi-tenant database architecture.
- [x] Tenant data isolation (ComplexId).
- [x] Independent Mercado Pago integration per tenant.
- [x] Clean URLs (`/[slug]`).
- [x] Complex registration and self-management.
- [x] Security audit and sanitization.

## Phase 2: Monetization & Core Features (IN PROGRESS)
- [ ] **Subscription Model**: Restrict field management until the initial platform fee is paid via Mercado Pago.
- [ ] **Inventory Management**: Allow complex owners to list rental items (balls, bibs, etc.).
- [ ] **Booking Add-ons**: Clients can select inventory items during the booking flow.

## Phase 3: Engagement & Scaling (UPCOMING)
- [ ] **WhatsApp Notifications**: Automatic booking confirmations and owner alerts.
- [ ] **Advanced Dashboards**: Financial reports and peak hour heatmaps for owners.
- [ ] **Custom Subdomains**: Support for `complex-name.platform.com`.
- [ ] **League & Tournament Module**: Tools for managing local tournaments and standings.

## Phase 4: Long-Term Ecosystem
- [ ] **Player App**: GPS-based court search and public match matchmaking.
- [ ] **Dynamic Pricing**: AI-driven price adjustments based on demand.
- [ ] **API Access**: Allow third-party integrations for massive complexes.
